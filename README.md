## Reflection

### What was the problem you were solving in this course?

The primary objective of this course was to design and implement software that helps academic advisors manage and access Computer Science course information. The system needed to load course data from a file, store it efficiently, and display it in alphanumeric order. The challenge involved selecting the most appropriate data structure to balance efficiency, scalability, and maintainability.

---

### How did you approach the problem?

I approached the problem by first analyzing three data structures: vector, hash table, and binary search tree (BST). I evaluated each structure based on:

- Time complexity for insertion and search
- Memory usage
- Ability to maintain sorted order
- Scalability for large data sets

Understanding data structures is critical because they directly affect the efficiency and performance of software systems. By comparing Big-O runtime and memory trade-offs, I was able to determine which structure best supported ordered output and efficient searching.

---

### How did you overcome roadblocks?

While completing the projects, I encountered challenges with:

- Parsing and validating file input
- Debugging pointer logic in the binary search tree
- Ensuring proper alphanumeric ordering of course IDs
- Managing segmentation faults during node insertion

I overcame these issues by breaking the problem into smaller components, testing incrementally, and carefully tracing program execution. Using structured debugging and reviewing algorithm logic helped reinforce my understanding of memory management in C++.

---

### How has this work expanded your approach to designing software?

This course reinforced the importance of selecting the correct data structure before implementation. Instead of immediately coding, I now analyze performance requirements and scalability first. I also consider time complexity and memory efficiency early in the design process.

This structured approach improves overall system performance and prevents costly redesign later.

---

### How has this project evolved the way you write maintainable and adaptable programs?

This project improved my coding practices in several key areas:

- Writing modular, reusable functions
- Separating data logic from user interaction
- Using clear naming conventions
- Commenting complex sections of code
- Designing adaptable structures that can scale

I now focus more intentionally on writing clean, organized, and maintainable code that can evolve as requirements change.
